Generators :

1.Generator classes are used to generate the identifier or primary key value for a persistent object while saving an object in database.

2.All hibernate generator classes implements org.hibernate.id.IdentifierGenerator interface, and overrides the generate(SessionImplementor,Object) method to generate the identifier or primary key value.

3.If we want our own user defined generator, then we should implement IdentiferGenerator interface and override the generate()

3. tag (which is sub element of tag) is used to configure generator class in mapping file.

Syntax :

@CODE_START@@HIBERNATE@ value @CODE_END@

Assigned :

1.assigned is default generator.

2.We if didn't mention any generator then hibernate takes assigned as a generator.

3.In this case, programmer is responsible to give primary key value.

4.This generator supports in all databases.

Syntax :

@CODE_START@@HIBERNATE@ < id name="employeeId" column"EMPNO"> < generator class="assigned"/> < /id>@CODE_END@

Note :This is the default generator class used by the hibernate, if we do not specify generator tag element under id element then hibernate by default assumes it as "assigned"

Sequence :

1.sequence is database dependent. MySql doesn't support sequence.

2.Hibernate is responsible to generate id value. Even if programmer passes the id value, but hibernate won't take the programmer given value.

3.Hibernate will execute the sequence and assigns the value to the object when inserting a new record.

4.We need to pass our sequence to class.

Syntax :

@CODE_START@@HIBERNATE@< id name="employeeId" column"EID"> < generator class="sequence"> < param class="sequence" >SequenceName < /param> < /generator> < /id>@CODE_END@

If we didn't pass any sequence then hibernate will take default sequence. The default sequence taken by hibernate is hibernate_sequence.

Increment :

1.Increment generator is database independent.

2.Increment generator will generate id value based on formula Max of id value + 1.

3.It there are no records in database, then increment will generate 1 as an id value.

Syntax :

@CODE_START@@HIBERNATE@< id name="employeeId" column"EMPNO"> < generator class="increment"/> < /id>@CODE_END@

Note :POJO class, Mapping and Configuration files are same as above application.

Identity :

1.Identity is database dependent. Oracle database doesn't support identity generator.

2.Identity generator is similar to increment generator, but the difference is, increment generator is database independent and hibernate uses a select operation for selecting max of id before inserting new record.

3.But in case of identity, no select operation will be performed to insert id value for new record by hibernate.

4.Here id value is generated by the database but not by hibernate.

Syntax :

@CODE_START@@HIBERNATE@< id name="employeeId" column"EMPNO"> < generator class="increment"/> < /id>@CODE_END@

Note :POJO class, Mapping and Configuration files are same as above application.

Oracle doesn't support identity. So we will get exception if we use oracle database.

hilo :

1.hilo is database independent.

2.hilo will insert 1 as a id value for first record. For the second record it will insert 32768 as id value. And from next record it will increment the old id value by 32768 and stores the new value.

3.To store the count of id values, hibernate will use a separate table known as hibernate_unique_value. default column name in this table is next_hi.

4.To create our own table and column, we need to pass 2 parameters for hilo generator.

Syntax :

@CODE_START@@HIBERNATE@< id name="employeeId" column="EID"> < generator class="hilo"> < param class="table" >tableName < /param> < param class="column" >columnName < /param> < /generator < /id>@CODE_END@

Note :POJO class, Mapping and Configuration files are same as above application.

native :

native doesn't have a separate generator class. It just uses other generator classes. identity, sequence, hilo

1.First it will check whether the database supports identity or not.

2.If database not support identity then it will go for sequence.

3.If database not support sequence then it will go for hilo.

Syntax :

@CODE_START@@HIBERNATE@< id name="employeeId" column="EID"> < generator class="native" /> < param name="sequence" >EMPNO_SEQ < /param> < param name="table" >HIGH_VAL_TABLE < /param> < param name="column" >HIGH_VAL_COLUMN < /param> < param name="max_lo" >60< /param> < /id>@CODE_END@

Note :POJO class, Mapping and Configuration files are same as above application.
