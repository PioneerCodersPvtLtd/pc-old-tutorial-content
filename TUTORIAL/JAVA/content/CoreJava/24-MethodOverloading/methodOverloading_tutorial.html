What is method overloading ?

If a class have multiple methods by same name but different parameters, it is known as Method Overloading.

There are two ways to overload the method in java

  • By changing number of arguments
  • By changing the data type
  • Method overloading is not possible by only changing the return type of the method

Why we need method overloading ?

If we have to perform only one operation, having same name of the methods increases the readability of the program.

Suppose that you have a class that can use calligraphy to draw various types of data (strings, integers, and so on) and that contains a method for drawing each data type. It is cumbersome to use a new name for each method-for example, drawString, drawInteger, drawFloat, and so on. you can use the same name for all the drawing methods but pass a different argument list to each method. Thus, the data drawing class might declare four methods named draw, each of which has a different parameter list.

Advantage of method overloading?

  • Method overloading increases the readability of the program.
  • It also gives you more flexibility while writing API.
  • Makes your API a lot simpler to use.
Example of Method Overloading.
public class DataArtist { public void draw(String s) { System.out.println("draw method of String "+s); } // type of arguments are different compare with above method. public void draw(int i) { System.out.println("draw method of int "+i); } // type of arguments are different compare with above method. public void draw(double f) { System.out.println("draw method of double "+f); } // list of arguments are different compare with above method. public void draw(int i, double f) { System.out.println("draw method of int and double "+f); } public static void main(String args[]){ DataArtist art = new DataArtist(); art.draw("Pioneer Coders"); art.draw(88); } }

Why Method Overloaing is not possible by changing the return type of method?

In java, method overloading is not possible by changing the return type of the method because there may occur ambiguity. Let's see how ambiguity may occur:

public class Addition { int sum(int a,int b){ System.out.println(a+b); } double sum(int a,int b){ System.out.println(a+b); } public static void main(String[] args){ Addition obj = new Addition(); int result=obj.sum(20,20); //in above statement, how can java determine which sum() method should be called. } }

Can we overload main() method?

Yes, by method overloading. You can have any number of main methods in a class by method overloading. Let's see the simple example:

Simple example on method overloading
public class MainOverloading { public static void main(int a){ System.out.println(a); } public static void main(String[] args){ System.out.println("main() method invoked"); main(12); } }
