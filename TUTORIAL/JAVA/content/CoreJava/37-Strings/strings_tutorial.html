What is String in java?

Normally, String is a sequence of character. But in java, string is an object that represents a sequence of characters. String class is used to create string object

Similarly to store the string data and to perform various operation on String data, we have three predefined classes they are:

String

StringBuffer

StringBuilder

String

String class object is immutable that means when we create an object of String class it never changes in the existing object

Example:

class StringEx { public static void main(String[] args){ String s = new String("pioneer coders"); String data=s.concat(" software"); System.out.println(data); } }

Note:here we can't change the object of string class

Create String object:

two ways to create String object:

  • By string literal
  • By new keyword

String Literal

String literal is created by using double quotes.

String s = "coders"

If we use above instruction, then string object will be created in String Constant pool area in method area memory.

Note: String Constant pool area objects are not eligible for garbage collection, Heap memory objects are eligible for Garbage collection. String constant pool area objects are destroyed when our java program is terminated or when our system is shutdown.

Each time you create a string literal, the JVM checks the string constant pool first. If the string already exists in the pool, a reference to the pooled instance is returned. If string doesn't exist in the pool, a new string instance is created and placed in the pool.

For example:

@CODE_START@@JAVA@ String s1 = "coders" String s2 = "coders" // will not create new instance @CODE_END@

In the above example only one object will be created. Firstly JVM will not find any string object with the value "Welcome" in string constant pool, so it will create a new object. After that it will find the string with the value "Welcome" in the pool, it will not create new object but will return the reference to the same instance.

By new keyword

String s = new String("coders");

If we use above instruction, then string object will created in heapmemory and in String constant pool area. In this context 's' variable is able to refer heap memory object
